Problem installing update due to proxy (problem with recent version)

I've been using Qiqqa on my work machine for a couple of years now. I posted a while back (year+) about authenticating to the proxy during log on, and that issue was resolved. Since then, everything seems to work just fine, including the auto update feature. It's a nice feature since it is a hassle to download the installer manually from my work machine. However, something changed in a recent release and the update fails. From the stack trace below, I think it is related to the same .NET proxy authentication process as the previous issue. It starts downloading, but fails before completion.
