Some general questions about tagging

Hi all,

1-TAGs
I have been working for two months with Qiqqa and so far I haven’t come up with a good way of classifying the information. May be I am not using tagging properly.
Before using Qiqqa I used Windows Explorer to classify the information contained in papers and articles. As everyone has done, I created a folder structure according to a high level conceptual classification of the topic I was studying. Later I copied each paper in each folder that this paper was related with. I call it “vertical classification”. The problem with this approach is that I couldn’t find easily very specific information (low level information).

Using Qiqqa I can create tags for very specific information and allows me finding the information along the whole library. I would call it “horizontal classification”. But so far I didn’t come up with a way of assigning tags that allows classifying the papers into a high level conceptual structure (vertical classification). This is also important when I backup the library because Qiqqa create a folder structure same as the list of tags.

I have upgraded to V44s. In the previous version, under the Tag Pane, in the left of each tag there was a small icon that allowed displaying all the tags associated with it. This feature doesn’t exist in V44s. I founded this feature quite useful.
How can I create Tags that allows classify the information in such vertical way?

As for me it would be very useful if it were possible to create a tree structure in the tag pane “High Level Tags”. As I read the papers and I assign tags I will have a list of “Low Level Tags”. Later I could copy or move these “Low Level Tags” on each of those “High Level Tags”.

2-.DISPLAY
I find quite pale the highlighted items. It can ́t distinguish easily from the rest of items. For example when I select a document and I scroll up or down, once I want to return to the selected document I can’t see it at first glance.
Is it possible to configure the intensity of the highlighted items?
Is it possible to assign colors to the status of the document in the same way as it is possible to do it with the emails in Microsoft Outlook using the categorize utility?

3-SNIFFER
Sometimes in the top windows the PDF documents appears too close and I would like to zoom out in order to select properly the whole title of the document.
How I can zoom in/out in the sniffer?
Is it possible to delete a document from the sniffer pane? If I can’t find BibTex data may be I would like to remove it from the library.

4-ANNOTATION BOX
Once I have created an annotation box, can I extend its size dynamically to include another part of the paragraph?

    TCS over 8 years ago
    In regards to #1

    At first, I found the lack of "vertical" tagging as something unnerving; but now - I don't really notice; and have found some pretty powerful navigation mechanisms that were not possible only having "vertical" classification.

    What I have found useful to develop a scheme where I precede my tags with special characters. These special characters sort first... and therefore show at the top of my list of tags.

    For my highest level classification scheme (primary, secondary, tertiary to the topic - supporting topic, etc) I use hash tags.

    For references to key primary themes, I use double underscore

    For journal ranking, or set, I use underscore (I have organized the journals into tiers and “collections” for my field, and assign tags to these levels of journals)

    For references to other key papers - I precede these with Tildes.

    So with this, I can select top journal, primary to topic, referencing key paper, containing Topic and Topic 2... and get a list of papers that fit that combination.

    When I feel I need to create a new class - I'll just add another character.

    Though I'd still like to see some sort of tag classification built in, the above scheme does work nicely.

    In regards to #2 - You can set the opacity of the highlighting in the configuration.

    In regards to #3 - on my version it adjusts to the screen size I give it, and it automatically fits the page.

    In regards to #4 - Yes... just click on the box (the text are) hold down your shift key, and move your mouse to resize.

    ...Hope this is helpful.

    TCS over 8 years ago
    I can't help you with the specifics for how you should arrange you tagging; there are field norms and tacit personal preferences that I do not /cannot know.

    In your last message you expressed issues with time (#1), and potentiality for error (#2) - and we started with A) not having a way of doing it.

    If your real issue is Z) I want Qiqqa to change, I can't help there. All I can do is try to help offer my own experiences with working with Qiqqa's current functionality.

    The best suggestion that I can make is try to work with it a little more, and focus think in terms of tagging and tag search rather than filesystem/directory structures. Notice that by selecting tags your are really doing a search. To navigate is to create a combination of tags that constrain your list being displayed in the "main pane". To navigate you use the left pane - to know where you are current at, you see the small pane above the main list which shows the combination of constraints (tag1 AND tag2, etc). So, for instance, if I wanted to display papers that discussed the Choral works of Bach and Handel (again, this i not my field) then I'd have to have tagged my documents with Composer, and Instruments (or whatever classification name is appropriate)... then I selected Choral in the left box... then from the resulting list, while holding down the Ctrl key, I'd click on Bach and Handel.
