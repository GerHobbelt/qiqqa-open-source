    i am pleased to report that I have worked out a partial fix for the profile problem.

    When I went into "tools"/"configuration"/"support functions" I found that the Qiqqa temp directory did not exist (C:\Users\\AppData\Local\Temp\Qiqqa\) so went to the correct location and created a folder Qiqqa. Now the autotags work without creating errors.

    Suprisingly my qiqqa v79 on another computer also had this folder missing, but the software was running happily without it!

    In the previous version libraries were deleted via the web interface which no longer exists in v80.

    Ger Hobbelt 12 months ago
    When you run into the problem of a broken (non-working) qiqqa search index and regenerating the index doesn't recover from that situation, chances are that you have exactly the same problem that i've had with qiqqa for a couple of years.

    The cause of that problem is having a library which includes some broken pdfs (broken in the perception of the qiqqa code). You MAY be able to dig those pdfs out by going through the qiqqa log file and looking for exceptions. (I am assuming you're running qiqqa v79 or v80)

    With those qiqqa releases the only available solution then is to find those pdfs and delete them. That's a tough job for any library of size. Pdfs are stored named by their content hash and I dont know/recall now if qiqqa v79 reports the hash/fingerprint of the offending pdfs in the logfile - all i know for sure is that i added a lot of those fingerprints to log reports throughout the code.

    v81 should not suffer from that issue and recreate your search index, but I would advise you VERY STRONGLY to make a library backup before trying that one. My v81 build is meant as a proof of "can anyone with the tools create a working qiqqa installer?" and also given ongoing work since then i would recite "backup backup backup" as a tantrum. That being said, iff you backup, you can install v81 if you cant resolve your issues otherwise and see if that one does fix your troubles. The advamtage of v81 is tgat it yaks a lot more in the logfile but most important: any crash/fail opportunity that i could fix or at least report-and-ignore is available in that build.

    Meanwhile this also means I'll have to do a release / build of the bleeding edge which should be backwards compat in some way (github issue 43 off the top of my head) to help diagnose and fix these issues.

    (you can get 'broken' / seriously corrupted pdfs into your library pretty easily, not qiqqa's fault. Trouble is qiqqa isnt very forgiving when it runs into bad input plus the libs it uses had some horrible pdf bugs too - see the github issues up to about number 20.

    Do me a favor and file this at the github repo: it doesnt magickally ;-) solve your troubles but it makes it easier for me or other devs to track and follow up as time allows.
