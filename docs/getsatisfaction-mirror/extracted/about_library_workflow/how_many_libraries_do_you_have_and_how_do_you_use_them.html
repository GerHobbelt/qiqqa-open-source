About library workflow - how many libraries do you have and how do you use them?

I am a new user and have not yet quite established my workflow in using libraries. I've gathered now about 25+ references for an article I'm writing and named the library according to that article, so I'll know which I've used. If I use this logic, then I should have a separate reserve library of ALL articles I've ever read or referenced.
But if I understood correctly, QIqqa copies the PDFs between libraries, so annotations to one version will not show up on another version in another library? I don't like to hunt for duplicate versions.

How do you keep track, which references you've used in which articles in Qiqqa? Tabs?

How do you use libraries? What is your logic in dividing PDFs between different libraries? Is it by-article, by-theme, by-what? Or do you just have one main library that you work in, and then some collaboration libraries?

    erladams over 6 years ago
    I have four libraries and tend to divide libraries by project or work site. Currently I have:
    - Thesis: My first library (approx 700 articles)
    - Preliminary Exams: This project was actually a meta-analysis, so I needed to keep careful track of the articles I found for this particular project; I didn't want to risk mixing it up with/confusing it with my thesis work. (Approx 1,000 articles)
    - PH: This designates a lab I work in and contains lit for two different projects with that group. Within the library I sort papers for specific projects by tag. All articles relevant to project 1 are tagged "Youth" and all articles for project 2 are tagged "SID." This makes it easy to only display the articles of a given project. This is a different area of research than my thesis and prelims, so there is little content overlap. (Approx 400 articles).
    - Dev: This was a small library (approx 40 articles) I generated for an end-of-semester project in a specific class. The project and literature isn't really related to anything else I do, so I didn't want it cluttering things up.

    My thesis library is the most inline with my mentor's research and so when I'm working on various publications for her lab I tend to pull cites from there, and I tend to store extra citations for her lab there as well. I familiar enough with the lit in each library that if I need a thesis citation for my prelims I can switch over and pull that paper in pretty quickly.

    Amba Morton over 5 years ago
    Three years in....my own method is now to throw all my pdfs with the correct bibliographic info into an endnote group for a thesis chapter, and then import into Qiqqa into a single library dedicated to that chapter.

    I add to it as I go, painstakingly putting into Endnote, then into Qiqqa - this means every damn thing I cite is correctly indexed and I need not go back and check. I don't find the bibtex and citation features of Qiqqa reliable enough and I have Endnote free through uni, so that works for me.

    And I use Microsoft Outliner now for all my writing, because Outliner WORKS with Endnote and is almost exactly like Scrivener - and can handle long documents (which regular Word can't do).

    I tried Zotero, I tried Mendeley, and I tried ReadCube - i tried Scrivener. None of this worked.

    I also use Calibre for housing my epubs, djvu, mobi and pdfs in a page view format, in libraries organised roughly by subjects I am covering in my thesis. So I open them and the cover pops up. As I have about 10,000 books in electronic format, and my first idea was to dump all these into Qiqqa (MISTAKE!) this lets me browse what I have and then decide what I will use, before putting into Endnote and Qiqqa.

    I do also have several Qiqqa libraries (big ones) in which most of the bibtex is there, and I can search them for information across board for anything I am researching. Annotation reports are the other thing I use it for, and expedition, but not brainstorming as its too hard to backtrack it and control this feature.

    I used to maintain many other Qiqqa libraries and check all the bibtex matched and so forth but it became too much work to put all the bibtex in manually (scholar couldn't find it all), and of course, Qiqqa needs bibtex to work optimally. It also meant slow loading. So now I have a handful of libraries which are 'clean' and fully searchable, and not linked to particular projects, and then the thesis chapter libraries. This workflow makes sense to me, hope it helps you:-)

    It is called Writing Outliner, I messed up the name, sorry. Link to site: http://writingoutliner.com/

    Another thing I recently figured out is how to force Word to handle a long document using the Outliner add-in with Endnote. May be useful (for Qiqqa's citation features and for anyone using Word for a thesis). What I do now is put all the chapters into Outliner, and set the bibliographic style in Endnote to one which has no bibliography. Then you update citations, and of course, Endnote generates a folder for that document, the content of which is dropped into a 'group'. Same for every subsequent chapter, dropping the contents of the folder into the same group. Then at the end, select all the contents and dump into a new document. Choose the same style, but the one which HAS a bibliography. And voila, all your citations for the entire project are there. Combine documents, convert to pdf = thesis in Word.
