    Mark over 8 years ago
    From a database and cloud storage point of view I can see how uniquely generated filesnames can be easier to handle.

    Maybe an easy workaround is to provide an export function, which exports all pdf's to a folder with the original filename, or better yet a generated filename based on the metadata e.g. %JOURNAL% %VOL% %YEAR% (%AUTHOR%, %TITLE%).PDF
    This can then be backed up to DVD and give the user peace of mind.

    However, the current situation is terrible from a user perspective. Even whilst with Qiqqa you are not "locked-in", the user will be screwed if his/her HDD crashes (and tries recovery) or the Qiqqa database becomes corrupt or operating systems/Qiqqa versions change. In theory the cloud storage is your backup, but it isn't always feasible to keep everything online (cost, copyright, confidentiality restrictions, internet bandwidths).

    For now, I must maintain two duplicates, a normal filename-based folder library and qiqqa's library on my HDD (needs 2x the storage, and 2x the admin). :(
