Goodbye Qiqqa; it's you not me

I finally had it with all the bugs and problems in this software. I loaded an easy pdf which locked up a major research library. This was the last straw. Qiqqa is garbage...a dead program.

No technical support, no updates (the last was a joke, didn't enhanse or fix anything...data waste), thanks for bilking me out of my storage fees.

For those looking for a replacement, Zotero has almost all the advanced features you "get" with Qiqqa. On comparison charts it seems Zotero is limited, but as a Firefox open source program, it has lots of enhancements (add ons) which make it very comparable with the features you look for in Qiqqa. It gets almost monthly updates (the new full version, 5, is now in beta). Zotero is produced for free by a major university, a gold standard. Stop suffering through this terrible Qiqqa software, make the switch. Importing my library took about 20 minutes.

    • Dennis Crawford over 3 years ago
      I have about 1k citations and pdfs (some quite large) across half a dozen libraries (in Qiqqa). I used the export function in Qiqqa to import my library. The whole process took about an hour (mostly because i was trying to figure out what I was doing and play with features, etc.). Not everything imported perfectly, but I've read that's the norm. It was otherwise painless. I'm writing a research design (my first big project since switching) and am kicking myself for not switching earlier. Zotero has a much easier citation workflow and is easier to navigate. Zero bugs so far.

      Zotero handles libraries differently than Qiqqa. Instead of organizing pdfs by individual library, I have one master library with subfolders broken into my research, courses, etc. Citations can be copied across subfolder but are really just one file. So, I don't have to worry about having to re-edit a pdf in multiple libraries. It's one pdf shared across libraries. This alone makes a surprising difference. Some other surprising features (copied from a recent email to a research partner):

      - Robust importing architecture. Unlike some other programs I tried (Qiqqa, Mendeley, ReadCube) I have yet to run into problems downloading pdfs from JSTOR, Google Scholar, etc. The other platforms were only reliable about 1/2 of the time. Zotero also does a good job of finding citations for pdfs I already own. Mendely sucked at this function. ReadCube did OK, but with a lot of user help.
      - Lacking an internal pdf reader, I’m free to use Nitro Pro which is highly capable at editing, notating, and running optical character recognition (OCR) scans. I’m not tied to more limited pdf reading software (ReadCube, for instance, won’t let you use a stylus to mark up a document nor highlight text if the pdf isn’t OCRed)
      - I downloaded a script which pulls my highlights and notations out of a pdf and creates a note file which can be copied to MS OneNote as an outline for writing papers. I prefer to have all my notes in one place vs. scattered across individual paper files.
      - By combining the OCR ability of Nitro Pro and the search functions of Zotero, every word in my library is searchable (“I need articles that mention, ‘education communities’. Ah here they are.”)
      - I discovered a way to analyze articles (again with OCR) to produce word clouds and brainstorm linkages.
      - There is a plugin which allows me to “check out” pdfs to my tablet. Zotero moves the articles to my dropbox, where I can access them from my tablet to take notes on, read, etc., then I just check them back into Zotero and it automatically pulls them out of dropbox, complete with annotations. You’ll see in my screenshot how this plugin created the files “Tablet Files” and “Tablet Files (modified)”. The program tags my checked out pdfs as being on the tablet. Once it sees I’ve been working on a pdf, it changes the tag to “modified” so it’s easy to keep track of what I’ve already done work on. No need to hassle with separate apps on my tablet.

      More than you asked for, but I was really bent on Qiqqa.

      //Dennis//

    • Dennis Crawford over 3 years ago
      The user guide page on Zotero simple and well written (https://www.zotero.org/support/). From there, it was easy to figure everything out. The plug-ins are just like in Firefox or Chrome where there is essentially a store (for some) and you just download the script, really easy.

      Note there are two different ways to get Zotero (when version 5 comes out of beta, there will only be one). You can either download the "standalone" which is its own separate program (like Qiqqa), or download it as a firefox program which runs in your browser. After a little research, I downloaded both. Having both is no big deal as they can't be open simultaneously and they connect (through something called a "translator") seamlessly. According to the internet, the browser-based version is slightly better at pulling down files from library databases. I haven't really noticed a difference. I tend to use the standalone more.

      As a side note, when you go to the Zotero website, you can find release notes on updates. They release new features and fixes almost every month (or so). The last release was only at the end of January and like I mentioned, the new version (5) is in beta testing. From what I can tell, Qiqqa really hadn't had any work done in a few years (only a token update which did nothing back in November), and their Android app was a joke; not touched since about 2012!

    • Dennis Crawford over 3 years ago
      Just about everything you need to know is on the Zotero support page (https://www.zotero.org/support/). Pdf file management is more straight forward than in Qiqqa. I have a watch folder that I can import from, but most new pdfs are just imported directly into Zotero via the plug-in. These pdfs can be stored anywhere you choose, but typically in a Zotero folder on the hard drive by default.

      For OCR, when you open a pdf in Nitro, if it's not already searchable, I get a pop-up (I don't remember if I had to set this to happen or if it always does it) asking if I want to OCR the doc. I can either click the button on the pop up or use the OCR button on the toolbar (I prefer this method because the popup will default to search plus edit which takes longer, from the toolbar, you can just OCR for search). Once the document is done being scanned, just click save and that's it. Zotero can now read the whole thing (within the limits of any OCR program). The key is to select the little drop down box in the Zotero search box and select "everything" (vs. author, etc.). It will pull up everything with that term in the folder you have displayed. Then you open the pdf and search again. I'll admit, it doesn't take you right to the term and sort as well as Qiqqa did, but it still works.

      The plug-in for brainstorms, etc. (disclaimer, I have yet to use this in Zotero and never found it practical in Qiqqa) is a plug-in located here: http://papermachines.org/
